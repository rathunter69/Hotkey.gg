◆ Formulas II · drill

Kill the hardcodes — make it roll forward

A hardcode freezes yesterday — relink it (=), then bulk-rename the version tag (Ctrl+H) so the whole page reads v2.
You inherited a five-year operating build where someone typed the answers: growth, gross profit, EBITDA, AND the margin are all hardcoded numbers — right today, silently wrong the moment an input moves. Resurrect every derived row as a live formula, fill it across, and prove nothing drifted. This is versioning discipline: v2 must survive new numbers without a rebuild.
train this drill → par 32 keystrokes · keyboard only · free
The keys
C5:F5 growth → =C4/B4-1, fill · B7:F7 GP → =B4+B6 · B9:F9 EBITDA → =B7+B8 · B10:F10 margin → =B9/B4 · stamp it v2 (ctrl+h v1 → v2) · costs carry their sign — subtotals are clean sums
Step by step
  1. growth row is typed · C5: =C4/B4-1 ↵ · select C5:F5, ctrl+r
  2. gross profit is typed · B7: =B4+B6 ↵ — COGS is negative; adding it IS subtracting · fill B7:F7
  3. EBITDA is typed · B9: =B7+B8 ↵ · fill B9:F9
  4. margin is typed · B10: =B9/B4 ↵ · fill B10:F10 — values must NOT move
  5. stamp it v2: ctrl+h · find v1 tab replace v2 ↵ — one pass renames every version tag on the page
